Ch1 - Functions and Polynomials

1A - Introduction to Functions

What is a function?

A function is a rule that receives an input (the x-value) and produces a single output (the y-value). In this definition, it is important to note the words "single output", as this will distinguish whether an equation represents a function or a relation. A straightforward way of distinguishing between a function and a relation is through the vertical line test.

Vertical-line test: any vertical line drawn on the graph of a function must only intercept the graph once.

y=x^2 passes the vertical line test and is a function

y^2=x fails the vertical line test and is not a function

Although we have noted that the left graph is a function, it is important to realise that it is also a relation. A relation is simply a set of inputs and outputs, and therefore all functions are relations.

Function Notation

Functions are most commonly denoted by the symbol f(x) (read as "f-of-x"), which means that an equation named f, has an input variable called x. Each function may be represented differently, such as; y, g(x), v(t), A(r), in these cases all that is changing is the name of the function and which symbol we choose to represent the input variable.

For a function, f(x)=5x^2 we can see that the function is dependent on x and replacing the variable x with a number or expression will accordingly replace all the x's in the function's rule.

It is also important, and may seem obvious, to remember that functions are exactly the same as "y", using functions just allows greater flexibility and provides more information, especially when keeping track of multiple formulas.

Types of Correspondence

Functions and relations can be classified according to the correspondence between inputs and outputs, resulting in four possible types. These types can each be viewed as the "number of x-values"-to-"number of y-values", and are as follows:

  • One-to-one ("one x-value to one y-value")
  • Many-to-one ("many x-values to one y-value")
  • One-to-many ("one x-values to many y-value")
  • Many-to-many ("many x-values to many y-value")

We see that one-to-one and many-to-one correspondences pass vertical line test and are therefore functions. Whilst one-to-many and many-to-many correspondences do not pass the vertical and are therefore not functions (but are relations)

1B - Domain and Range

Maximal/Implied Domain

Functions are often written without specifying a domain, in this case a function will take on its maximal or implied domain - this is the largest subset of all real numbers, R for which the function's rule will be defined. To simplify things, we only need to look for two cases in which the maximal domain of a function is restricted.

The maximal domain of any function must exclude:

  • Any input that would result in a denominator becoming zero (dividing by zero - resulting in the graph having a vertical asymptote and being discontinuous)
  • Any input that would result in a negative number in a square root (no real solutions - resulting in the graph not existing)

In the following sections, where our focus is on polynomial functions, the maximal or implied domains will always be R. Note that there may be additional cases to these two if the function appears in an application style question. A common example is when the question involves time as the input variable, since time cannot logically be negative this implies that the domain is restricted from negative numbers regardless of the function's rule.

1C - Linear Functions

The linear function is the simplest polynomial that we will encounter and is also the simplest function for us to graph - a straight line.

Gradient

The gradient, or slope, of a linear function represents the increase (or decrease) in the y-direction for every unit travelled in the x-direction. his can be calculated through the use of any two points on the line; (x_1,y_1 ) and (x_2,y_2 ), and the rise over run formula: m=((y_2−y_1 ))/(x_2−x_1 ). Also recall that regardless of which two points we have chosen on the linear function, the gradient will be the same since it is a straight line - however, this will not be the case for other functions, such as quadratics.

Other important formulas regarding the gradient involve parallel lines, which will share the same gradient, and perpendicular lines, which will evaluate to -1 when the gradients are multiplied.

  • For parallel lines: m_1=m_2
  • For perpendicular lines m_1 m_2=−1

It will also be useful to convert between the gradient, m, and the angle of the slope, by using m=tan^-1(θ), where θ is the angle between the line and the positive direction of the x-axis.

Equation of a Linear Function

The equation of a linear function should always be expressed in the gradient-intercept form: y=mx+c, where m is the gradient and c is the y-intercept. This is the form that examiners will expect answers to be in, unless specified otherwise.

Another form that will be useful for working out is y−y_1=m(x−x_1 ), this involves a point knowing a point on the line (x_1,y_1 ) and the gradient m. However, when using this form remember to rearrange to y=mx+c for the final answer.

1D - Quadratic Functions

Quadratics are second degree polynomials, that are graphed as parabolas. The quadratic is one of the most common polynomials in Mathematical Methods and are commonly expressed in the form ax^2+bx+c.

  • One-to-one ("one x-value to one y-value")
  • Many-to-one ("many x-values to one y-value")
  • One-to-many ("one x-values to many y-value")
  • Many-to-many ("many x-values to many y-value")

We see that one-to-one and many-to-one correspondences pass vertical line test and are therefore functions. Whilst one-to-many and many-to-many correspondences do not pass the vertical and are therefore not functions (but are relations)
